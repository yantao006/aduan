## 一、对标分析：Buffett vs 段永平

| 维度 | learnbuffett.com | learnDuanYongping（本项目） |
|------|-----------------|---------------------------|
| **核心内容** | 81 封股东信（英文，需翻译） | ~2,000+ 条博客/Q&A/访谈（中文原生） |
| **内容类型** | 单一：年度书信 | 多元：博客+雪球问答+访谈+企业文件 |
| **工作量最大** | 翻译 + 概念提取 | 结构化 + 概念提取 + 交叉链接 |
| **概念数** | 49（投资概念） | ~50（投资+商业+人生哲学） |
| **公司数** | 61（投资标的） | ~25（投资案例+经营企业） |
| **关键人物** | 7 | ~15（巴菲特/芒格/黄峥/陈明永/沈炜…） |
| **独特板块** | — | Stop Doing List / 投资案例时间线 / 语录检索 |
| **构建方式** | Claude Code 5 Agent 并行，1 下午翻译 | Hermes Agent（内置 delegate_task 并行），核心工作在**结构化而非翻译** |
| **输出** | Obsidian → 静态 HTML 网站 | 同样 Obsidian → 静态 HTML 网站 |

**关键差异**：段永平资料是中文原生，省去了翻译环节，但资料分散度更高、格式不统一。核心挑战在于**统一格式 + 提取概念 + 建立链接**。

## 五、分阶段执行计划

### 阶段 0：Markdown 化（✅ 已完成）

- [x] 7 篇 HTML → MD（已添加 YAML frontmatter）
- [ ] 2 本核心 PDF → MD（投资逻辑篇 + 商业逻辑篇，可用 marker-pdf 或 LLM 提取）
- [ ] 7 份次要 PDF → MD（按需，部分与现有 .md 重复）

### 阶段 1：格式标准化（预计 1-2 小时）

**目标**：给所有 .md 文件添加统一的 YAML frontmatter

```
---
title: "段永平博客：买股票就是买公司"
date: 2010-03-15
source: "网易博客"
source_url: "http://nteswjq.blog.163.com/..."
type: "blog"
tags: [价值投资, 能力圈]
concepts: [买股票就是买公司, 未来现金流折现]
companies: [Apple, 网易]
people: [巴菲特]
---
```

**做法**：Hermes Agent 批量处理（delegate_task 并行 5 Agent）
- 每批 100-150 篇
- Agent 读取原文 → 提取标题/日期/来源/关键词 → 写入 YAML 头 + 保留正文
- 人工抽检 5%

### 阶段 2：概念提取 + 卡片生成（预计 2-3 小时）

**目标**：从所有内容中提取 ~50 个投资概念，生成标准概念卡片

每个概念卡片格式（对标 learnbuffett.com）：
```markdown
---
title: "能力圈"
type: "concept"
category: "投资哲学"
first_mentioned: "2010-03-15"
mentioned_count: 87
related_concepts: [安全边际, 不懂不做, 集中投资]
related_companies: [Apple, 网易, 茅台]
related_people: [巴菲特, 芒格]
---

# 能力圈

## 段永平怎么说
> "知道自己能力圈有多大，往往比自己能力圈有多大要重要得多。"

## 核心要义
1. 只投自己真正懂的企业
2. 能力圈可以慢慢扩展，但不能跳出去
3. 不懂装懂是最大的风险

## 经典案例
- 网易：段永平懂游戏行业（自己做小霸王/步步高），在别人看不懂时买入
- Apple：深度使用产品 → 理解护城河 → 重仓

## 演变轨迹
- 2010-2012：在博客中频繁强调"不懂不做"
- 2013-2015：结合 Apple 投资案例展开论述
- 2018-2020：在雪球上多次用"能力圈"回答网友提问
- 2025：浙大演讲再次强调

## 引用来源
- [[2010-03-15 网易博客：买股票就是买公司]]
- [[2013-05-20 雪球：关于Apple]]
- [[2018-09-30 斯坦福对话]]
- （...共 87 次提及）

## 常见误解
- ❌ "能力圈就是要很窄" → 段永平：能力圈可以扩展
- ❌ "懂了就可以 all in" → 段永平：懂了只代表可以投
```

**做法**：Hermes Agent 用 delegate_task
- 5 个 Agent 并行，每个负责 ~10 个概念
- 每个 Agent 搜索所有文件的 `[[概念名]]` 引用 → 整理卡片

### 阶段 3：案例 + 企业 + 人物档案（预计 2 小时）

**目标**：生成 20 个案例档案 + 20 个企业档案 + 15 个人物页

参照阶段 2 的卡片格式，按投资案例/企业/人物分别生成。
关键：每个档案都要有**时间线 + 段永平原话引用 + 与其他概念的链接**。

### 阶段 4：交叉链接（预计 1-2 小时）

**目标**：在博客正文中标注 [[概念]] [[公司]] [[人物]]，实现双向链接

```markdown
# 2011-01-15 网易博客：关于Apple的投资逻辑

段永平在博客中说：我买 [[Apple]] 的逻辑很简单——
这是一个有强大[[护城河]]的公司...
这让我想起[[巴菲特]]说的...
当时很多人问我为什么不买[[Google]]，我的回答是：[[能力圈]]。
```

**做法**：
- Hermes Agent 逐篇处理（delegate_task 10 Agent 并行）
- 识别文中出现的概念/公司/人物 → 添加 [[wikilink]]
- 保留原文不改动（仅添加链接标记）

### 阶段 5：知识图谱数据生成（预计 1 小时）

**目标**：生成 JSON 格式的图谱数据（对标 learnbuffett.com 的 search-index.json）

```json

    {"id": "Apple", "type": "company", "count": 156, "group": "投资案例"},
    {"id": "网易", "type": "case", "count": 89, "group": "投资案例"}
  ],
  "links": [
    {"source": "能力圈", "target": "Apple", "weight": 12},
    {"source": "能力圈", "target": "不懂不做", "weight": 45},
    {"source": "Apple", "target": "护城河", "weight": 23}
  ]

```

### 阶段 6：静态网站构建（预计 2-3 小时）

**对标 learnbuffett.com 的功能**：

| 功能 | 实现方式 |
|------|----------|
| 首页总览 | 概念 TOP 15 + 案例 TOP 15 + 搜索框 |
| 知识图谱 | D3.js 力导向图，可缩放/拖拽/筛选 |
| 概念索引 | 所有 50 个概念，按类别分组，标注出现次数 |
| 公司索引 | 所有 20 个公司，按关系分类 |
| 人物索引 | 所有 15 个人物，标注被引用次数 |
| 全文搜索 | 静态 JSON 索引 + 前端搜索（同 learnbuffett） |
| 博客原文 | 按年份分组的时间线浏览 |
| 响应式 | 移动端侧边栏折叠 |

**可选增强**：
- AI 问答（"段永平下午茶"）— 用 Hermes Agent 或 LLM API
- 13F 持仓数据可视化
- RSS/更新通知

### 阶段 7：部署上线

- 平台：Cloudflare Pages（免费，对标 buffett-letters-eir.pages.dev）
- 域名：可选 `learnDuanYongping.com` 或类似
